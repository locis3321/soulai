PART 01 · 现状诊断

交互缺主流习惯
视觉缺宗教重量

经过对现有 web / iOS / Android 三端视图与五大模块入口的快速审视，问题集中在四个维度——它们互相纠缠：没有主流操作就没有日常留存，没有视觉重量就没有仪式信任，没有仪式节奏就没有神秘学的"慢"，没有内容克制就没有边界。下面是四张诊断卡。

01 · 操作主流性

操作不符合主流 App 习惯

动画、点击、按钮均偏离 iOS / Android 用户已经内化的肌肉记忆，导致首次用户的"陌生感"被转化为"不信任"。

  • 缺底部 tab 导航，跨模块切换依靠左上箭头
  • CTA 没有固定位置，靠用户滚动寻找
  • 缺标准 loading / 错误 / 空态反馈
  • 滑动手势与系统手势冲突（如边缘返回被拦截）
02 · 视觉宗教性

视觉缺宗教 / 灵性元素

配色与排版完全是 SaaS 通用风，与"问卜 / 占星 / 疗愈"的灵性产品定位严重错位。视觉重量不足以承载用户的期待。

  • 缺东方禅意母题（朱砂 / 水墨 / 印章 / 山形）
  • 字体多为系统默认 sans，无立骨感
  • 图标使用通用 emoji，缺乏仪式符号
  • 留白不足，信息密度堆叠像"功能墙"
03 · 仪式节奏

问卜缺乏"仪式过程感"

用户期待的是一次"小型仪轨"，实际拿到的是"立即出结果"。缺失的过程感直接削弱了产品的神秘学价值。

  • 无问卜前的"静心"环节
  • 解读直接以纯文本弹出，无字渐显 / 笔触缓画
  • 抽牌 / 起卦没有物理感的反馈（牌背翻转、笔触）
  • 无仪式音效 / 触感反馈
04 · 内容克制

解读与付费缺乏边界

部分文案接近"断言式预言"，且付费墙与"消灾 / 解锁诅咒"暗示挂钩，与产品合规边界冲突。

  • 解读语气偏断言式（如"你注定..."）
  • 付费墙暗示"消除厄运 / 解锁诅咒"
  • 缺统一的 Disclaimer（自省 / 娱乐 / 不构成建议）
  • 推送频率与文案语气偏恐惧 / 依赖驱动

PART 02 · 定位重述

东西方融合
五大模块共一仪轨

核心张力：神秘学的多元 vs 体验的一致。我们的解法是不让五大模块争夺同一个入口——它们共享一套仪式外壳，而入口本身成为用户的主动选择。

「SoulAI 是东西方融合的身心灵修行 App：保留塔罗 / 紫微 / 八字 / 占星 / 数字命理 五大模块，以东方禅意（朱砂 / 水墨 / 留白）作为统一的视觉外壳，让每一次问卜都成为一次有节奏的仪轨——而 App 本身用主流的操作习惯承接你的日常。

P1 · Ritual First

仪式感优先

每一次问卜都被设计成小型仪轨——静心、起卦、解读、沉淀。让神秘学的"慢"成为产品差异化的护城河，而不是被 AI 抹平的"快速答案"。

P2 · Mainstream UX

主流操作为底

底部 tab / 标准 CTA / 系统返回 / loading 反馈——所有"看不见的"操作都遵循主流 App 的肌肉记忆。仪式感只发生在内容层，不发生在交互层。

P3 · Mystical Bone

神秘学为骨

东方禅意的视觉系统（朱砂 / 水墨 / 留白 / 印章 / 山形）作为骨架，而不是作为"皮肤"。五大模块共用同一套母题，但解释语言各依其传统。
